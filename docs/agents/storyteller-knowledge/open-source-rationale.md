# Why Open Source (from Fellowship)

Source: Michael Raspuzzi's fellowship resources, April 2026.

## 5 reasons the fellowship recommends open source

1. **You own what you build.** Trad ed owns everything you make (need to license it back). Worldwide owns nothing — team members have full rights after.

2. **Clean exit options.** Keep building solo, fork it, work with a teammate, work with Worldwide — total freedom.

3. **Build a movement.** Best engineers get $1M+/yr at top AI companies. No clear incentive for them to build ed tools. Open source lets you recruit builders and engineers fractionally to close the gap.

4. **Increases velocity between fellows.** One team builds on Agent SDK → another team uses it. One team builds the best skills graph → others benefit. Proven in SF hacks.

5. **Choose your own license.** Fellowship recommends Apache 2.0.

## License recommendations

| License | Key trait | Best for |
|---|---|---|
| **Apache 2.0** (recommended) | Permissive + patent grant | Enterprise safety, patentable tech |
| MIT | Maximum permissive, no patent grant | Maximum adoption, don't care about downstream |
| GPLv3 | Copyleft — derivatives must stay open | Keeping ecosystem open, community contributions |
| BSD 3-Clause | Like MIT + no-endorsement clause | Protecting brand from forks |
| GPLv2 | Copyleft, less restrictive than v3 | Hardware contexts, broader corporate compatibility |
